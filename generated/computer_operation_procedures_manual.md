# Computer Operation Procedures Manual

## 1. Purpose

This Computer Operation Procedures Manual (COPM) provides information and operating instructions related to the operation of the computer system. The intended users are the operating staff of the relevant section. This manual is not a complete replacement of formal technical publications but provides installation-specific details.

## 2. Scope

This document serves as a guide for operating the system, addressing tasks monitored by the administrator to ensure proper system function. It covers software and hardware configurations, system environment details, application configurations, server and workstation information, and backup services.

## 5. System Summary

### 5.1 Objective

The Licensing Self-Certification Portal (LSCP) aims to provide an electronic platform for site inspection and monitoring personnel. It allows for efficient management, review of records, and submission of application forms and electronic documents.

### 5.2 System Architecture

The LSCP architecture includes both on-premise (WKGO) and cloud-based (GCIS) components.

*   **On-Premise (WKGO):**
    *   Internal network behind a firewall with NAT.
    *   Production, UAT, and DEV environments for internal users.
    *   Reverse proxy server with load balancing.
*   **Government Cloud Infrastructure Services (GCIS):**
    *   Internet DMZ (iDMZ), Trusted Zone, and Gnet DMZ (gDMZ).
    *   Reverse proxy server and Web Application Firewall (WAF) in front of iDMZ for security.
*   **External Users:** Access the system via the internet through the LSCP Web Application.
*   **Internal Users:** Access the system from the BD intranet through the Departmental Portal (OSDP).

**System Components:**

*   **External Application Server:** Serves static web content and proxies backend APIs.
*   **External Web Server:** Hosts the backend API using ExpressJS for business logic and database operations.
*   **BD Web Servers:** Similar to external web servers but deployed in different zones for internal BD users.
*   **Database Management Servers:** Microsoft SQL Server database engine.
*   **iAM Smart:** Provides secure login and user information retrieval.
*   **Departmental Portal:** Web service for BD user identity verification.
*   **SMTPX:** Used for sending login one-time passwords (OTP) and email notifications.
*   **MWMS:** Provides data of AP/RSE/RGE/AS, interfaced to BDSCS through SFTP.
*   **BCIS:** Provides address data and BD case data, synced to BDSCS through SQL queries.
