# Computer Operation Procedures Manual

## Purpose

This Computer Operation Procedures Manual (COPM) provides information and operating instructions for the Licensing Self-Certification Portal (LSCP) system. It is intended for use by operating staff to monitor and ensure the system functions properly. This manual supplements, but does not replace, formal technical publications from the manufacturer.

## Scope

This document serves as a guide for operating the system, outlining tasks for administrators to monitor the system's proper functioning. It details software and hardware configurations, system environments, application configurations, server and workstation information, and backup services.

## 5. System Summary

### 5.1 Objective

The LSCP aims to provide an electronic platform for site inspection and monitoring personnel to manage and review records. It also enables applicants, Authorized Persons (AP), Registered Structural Engineers (RSE), and users from the Social Welfare Department (SWD) and Education Bureau (EDB) to submit application forms and documents electronically. The system serves as a centralized repository for all applications and supporting documents.

### 5.2 System Architecture

The LSCP architecture includes both on-premise (WKGO) and Government Cloud Infrastructure Services (GCIS) deployments.

*   **On-Premise (WKGO):** Behind an internal firewall with NAT, separating Production, UAT, and DEV environments for internal users. A reverse proxy server with load balancing enhances security and distributes requests to frontend web servers.
*   **GCIS:** Comprises Internet DMZ (iDMZ), Trusted Zone, and Gnet DMZ (gDMZ). Each DMZ includes a reverse proxy server and a Web Application Firewall (WAF) for enhanced security.
*   External users access the system via the internet through the LSCP Web Application, which interacts with the Application Server through the reverse proxy.
*   Internal users access the system from the BD intranet through the Departmental Portal (OSDP), connecting to BD Web Servers in the Trusted Zone.
*   The system includes Log Servers, File Servers, Database Servers (Microsoft SQL Server), vCenter Servers, and Backup Servers.

### 5.3 System Functions

Key system functions include:

*   User authentication (with password and iAM Smart)
*   Single Sign-On (SSO) for BD, EDB, and SWD users
*   Document preview and printing
*   Document upload with virus scanning
*   Management statistics and reports generation
*   Electronic submission and processing of applications
*   Electronic tracking of applications
*   Centralized data repository
*   Search and capture capabilities
*   Handling new and alteration applications
*   Handling Self-Certification Submissions
*   Handling Periodic Inspection for CCC
*   Handling PTS for TPPE
