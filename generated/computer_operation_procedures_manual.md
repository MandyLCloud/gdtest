# Computer Operation Procedures Manual

## Purpose

This manual provides information and operating instructions related to the operation of the computer system for the Licensing Self-Certification Portal (LSCP) of the Buildings Department (BD). It is intended for operating staff.

## Scope

This document serves as a guide for operating the system, addressing tasks that need monitoring by the administrator to ensure proper system functionality. It covers software and hardware configurations, system environment details, and backup services.

### Definitions

None.

### Conventions

The following acronyms are used:

*   **BD:** Buildings Department
*   **LSCP:** Licensing Self-Certification Portal
*   **DMZ:** Demilitarized Zone
*   **SAN:** Storage Area Network
*   **VM:** Virtual Machine
*   **ITU:** Information Technology Unit
*   **WKGO:** West Kowloon Government Offices

## System Summary

### Objective

The LSCP aims to provide an electronic platform for site inspection and monitoring personnel to manage and review records effectively. It also allows applicants/Authorized Persons (AP)/Registered Structure Engineers (RSE) and users in Social Welfare Department (SWD) and Education Bureau (EDB) to submit application forms and electronic documents online.

### System Architecture

The LSCP architecture includes both on-premise (WKGO) and Government Cloud Infrastructure Services (GCIS) deployments.

*   **On-Premise:** Behind an internal firewall with NAT, separated into Production, UAT, and DEV environments for internal users. A reverse proxy server with load balancing enhances security.
*   **GCIS:** Divided into Internet DMZ (iDMZ), Trusted Zone, and Gnet DMZ (gDMZ), with reverse proxy servers and Web Application Firewalls (WAF) for increased security.

External users access the system through the internet, interacting with the Application Server via the reverse proxy. The Application Server hosts static web interface files. The system logic is processed by the Web Servers in the Trusted Zone, which interact with the Database Servers (Microsoft SQL Servers) and file storage.

Internal users access the system via the Departmental Portal (OSDP) through the BD intranet, connecting to the BD Web Servers in the Trusted Zone.
